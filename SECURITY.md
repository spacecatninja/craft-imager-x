# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 5.x     | ✓         |
| 4.x     | ✓         |
| < 4.0   | ✗         |

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues.

Instead, send an email to hi@spacecat.ninja. You should receive a response within 5 business days. 
If for some reason you do not, please follow up to ensure the original message was received.

Please include as much of the following information as possible to help understand and resolve the issue:

- Type of vulnerability
- The file(s) and line number(s) involved
- Steps to reproduce
- Proof-of-concept or exploit code (if possible)
- Impact and how an attacker might exploit it

## Disclosure Policy

Once a vulnerability is received, we will:

1. Confirm receipt within 5 business days
2. Investigate and determine the impact and severity
3. Aim to release a fix within 60 days — as a solo-maintained project this may vary depending on complexity, but critical issues will be prioritised
4. Publish a security advisory on GitHub once a fix is available
5. Credit the reporter in the advisory unless anonymity is requested
